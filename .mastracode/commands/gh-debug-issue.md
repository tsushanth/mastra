This command has been deprecated and replaced with `/understand-issue`.

The reason: `/gh-debug-issue` mixed issue understanding, reproduction planning, test creation, and fixing into one long flow. The new `/understand-issue` command owns issue investigation and diagnosis, builds context first, and can receive `/gh-triage` handoff context through `--working-file` when needed.

For PR review, see `/understand-pr`.

Please ask the user to run `/understand-issue` instead.
