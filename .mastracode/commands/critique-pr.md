This command has been deprecated and replaced with `/understand-pr`.

The reason: `/critique-pr` encouraged offloading critical thinking to agents without truly understanding the PR, the code areas it impacted, the architecture, or history. The new `/understand-pr` command improves on this by making PR review collaborative and archaeological — it builds deep historical context, traces architecture, and works with the reviewer to form opinions grounded in evidence rather than generating review comments from a surface-level diff read.

For issue investigation, see `/understand-issue`.

Please ask the user to run `/understand-pr` instead.
